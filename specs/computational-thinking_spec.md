# Course Specification: Computational Thinking

## Overview
- **Course Title**: Computational Thinking (التفكير الحاسوبي)
- **Course Slug**: computational-thinking
- **Chapters**: Multiple (Chapter 1 ready, more chapters pending)
- **Current Scope**: Chapter 1 — Computational Thinking

## Transformation Goal
After completing this course, university students will be able to:
- Apply computational thinking principles to break down complex problems
- Use logical and algorithmic thinking to design solutions
- Control algorithm execution through iteration and selection
- Examine and evaluate algorithms for correctness

## Audience & Context
- **Target Audience**: University undergraduate CS/IT students
- **Prior Knowledge**: Beginner level — first exposure to computational thinking concepts
- **Language**: Bilingual Arabic/English (Arabic primary, English for technical terms)
- **LMS**: Moodle
- **SCORM Version**: 1.2

## Content Type Definitions

### انفوجرافيك (Infographic)
- Visual, scrollable infographic-style layout
- Rich graphics, icons, minimal text
- Used for: Learning objectives overview, learning roadmap

### اختبار قبلي / بعدي (Pre-test / Post-test)
- Quiz-style assessment
- Pre-test: Diagnostic (not graded, gauges prior knowledge)
- Post-test: Summative (graded, measures learning achievement)
- Both track via cmi.interactions

### محاضرة تفاعلية (Interactive Lecture)
- Slide-based content presentation
- Embedded interactions (click-to-reveal, hotspots, tabs)
- Audio narration support (optional)
- Key concepts highlighted with visual callouts

### محاضرة PDF (PDF Lecture)
- Downloadable PDF reference material
- Companion to interactive lecture (same topic)
- Clean, printable layout

### فيديو موشن (Motion Video)
- Animated explainer video (motion graphics)
- Max 10 minutes (NELC compliance)
- Subtitles in Arabic and English

### نشاط تفاعلي (Interactive Activity)
- Hands-on practice activities
- Uses interactive components: drag-drop, sorting, matching, simulations
- Immediate feedback on actions
- Gamification: points for completion

### نشاط - نقاش (Discussion Activity)
- Scenario-based critical thinking
- Present gotchas/edge cases
- Reflection prompts with text input
- Peer discussion encouraged (LMS forum link)

### واجب (Assignment)
- Structured task with rubric
- Algorithm analysis exercise
- Submit via LMS (link out)

### ملخص (Summary)
- Key takeaways from the module
- Visual recap of main concepts
- Links to next module

## Art Direction
- **Theme**: Technical Dark (code editor aesthetic)
- **Palette**: Dark backgrounds, syntax-highlighting accents (green, cyan, orange, purple)
- **Typography**: Monospace for code, clean sans-serif for content, Arabic: Noto Kufi Arabic
- **Visual Style**: Terminal/IDE inspired, circuit board patterns, algorithm flowcharts
- **Motion**: Subtle code-typing animations, data flow transitions

## Assessment Strategy
- **Pre-test**: 10 questions, diagnostic (not scored toward grade)
- **Interactive Activities**: Points-based gamification (engagement, not graded)
- **Post-test**: 15-20 questions, passing score 70%
- **Question Types**: MCQ, true/false, matching, ordering, fill-in-the-blank

## Technical Requirements
- SCORM 1.2 compatible
- Moodle tested
- Self-contained (no external CDN)
- RTL support for Arabic content
- WCAG 2.1 AA accessible
- Mobile responsive
- Offline capable

## NELC Compliance
- Video max 10 min per clip
- Min 2 interaction types per module (exceeded: 6+ types)
- Arabic RTL with 18px+ body text
- Self-contained packages
- Measurable learning objectives at every level
